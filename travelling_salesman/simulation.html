Solution of Travelling Salesman Problem using Brute Force

Given a list of cities and the distances between each pair of cities, what is the shortest possible route that visits each city exactly once and returns to the origin city?









Use the above sliders to adjust the position of the point to be inserted


Trivia

This problem is classified as NP-hard since it's time complexity is \( \mathcal{O} (n^2) \) (Along with a space complexity of \( \mathcal{O} (n) \)). The Brute-Force method looks at all possible permutations and determines the one with the least distance. Runtimes go into minutes when there are ~7 points and into hours when there are ~9 points. A lot of improvements exist, and computing the solution of this problem is of special interest to theorertical computer scientists and operational researchers.



Note:
  1. When there are 8+ points, then an initial lag is expected before the solving process starts
  2. With 10 points, an average time of 49 minutes was required to find the solution to the route
  3. Simulation may lag on lower-end computers and phones
